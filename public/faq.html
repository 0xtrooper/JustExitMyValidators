1. What does this tool do?

This tool recovers your node address using your mnemonic phrase and then derives all the associated validator private keys. This functionality allows you to stop or disable a validator even if you don't have direct access to the smartnode.

2. When should I use this tool?

This tool is designed for situations where you want to exit all validators and do not have access to your synced smartnode. Because it compromises all validator private keys, it should only be used if you intend to exit every validator.

3. Why do I only need a mnemonic, without node access?

The smartnode is built on a Hierarchical Deterministic (HD) wallet architecture. This means that all private keys are derived from a single mnemonic phrase. Essentially, your mnemonic acts as a master key, allowing you to recover every validator's private key without needing direct access to the node itself. For more details on HD wallets, check out this article.

4. Is it secure?

The tool requires access to your mnemonic, which means it gains full access to your node wallet and all associated validator keys. Only your withdrawal address remains safe. Additionally, to help mitigate risks, the tool is open source and designed to run locally, ensuring that your sensitive data remains on your own device.

5. How do I set a withdrawl address?

To enhance security, it is highly recommended that you set a primary withdrawal address if you haven't already done so. You can learn how to set one by following this guide.

6. How does it integrate with beaconcha.in?

The tool generates a signed exit message that you can broadcast using beaconcha.in, allowing you to start your validator's exit.
